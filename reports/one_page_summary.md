# ReproLab: One-page summary

Independent reproducibility audit of an AI-for-science benchmark.
Repo: https://github.com/maruyamakoju/reprolab (2026-07-03)

## What I audited

The **Matbench Discovery** leaderboard (Riebesell et al., *A framework to evaluate
machine learning crystal stability predictions*, Nature Machine Intelligence 7,
836–847, 2025; repo `janosh/matbench-discovery`, pinned commit `eaa7550`). The
leaderboard ranks machine-learning interatomic potentials by how well they classify
crystal stability on the WBM test set (~257k structures).

## What I checked

- **Layer A — metric recomputation:** do the published per-model metrics (F1, MAE,
  confusion counts) follow from the released prediction CSVs and the bundled ground
  truth? Recomputed two independent ways (from-scratch reimplementation + the
  upstream metric function).
- **Layer B — prediction regeneration:** can the released predictions themselves be
  regenerated by running the model? CHGNet, MACE-MP-0, and ORB v2 relaxing the same
  deterministic 500-structure WBM subset (seeded sample, ids committed), scored
  through the same Layer A path. Comparison thresholds were pre-registered before
  the first GPU smoke and reused for the model extensions.

## Main results

- **Layer A: 4/4 released models (CHGNet, SevenNet-0, MACE-MP-0, ORB v2) reproduce
  the official YAML metrics exactly** — every fraction to 3 decimals and every integer
  TP/FP/TN/FN count, on both audited subsets.
- **Layer B: regenerated CHGNet formation energies match the published predictions
  with median |Δ| = 0.03 meV/atom** (p95 = 0.07, max = 1.08; threshold was 10),
  across a 2023→2026 dependency gap (torch 1.11→2.11, ase 3.22→3.29).
  100% stability-classification agreement, zero flips; 500/500 structures relaxed
  with 0 failures in 9.3 minutes on one RTX 4090.
- **Layer B second model: regenerated MACE-MP-0 also passes the same n=500 smoke
  criteria** — median |Δ| = 0.03 meV/atom, 99.6% stability-classification agreement
  (2 flips), 500/500 relaxed with 0 failures and 500/500 convergence in 10.1 minutes
  on the same RTX 4090. The three large MACE outliers (max 216.65 meV/atom) are the
  same MP2020 correction-version drift structures isolated in the ground-truth audit,
  not a broad relaxation failure; the other flip is a 0.1 meV/atom threshold-boundary
  case.
- **Layer B third model: regenerated ORB v2 also passes at n=500** — median |Δ| =
  0.05 meV/atom, 100% stability-classification agreement, zero flips, 500/500
  relaxed with 0 failures and 500/500 convergence in 5.2 minutes on the same RTX
  4090. It reproduced only with the YAML `max_force: 0.02`; the upstream runner's
  default `force_max=0.05` failed a two-structure pre-smoke, so this is a concrete
  protocol-ambiguity finding.
- **Layer C (statistical audit): the leaderboard reports no uncertainty.** A paired
  bootstrap (exact joint-category design, B=2000, n=215,488) gives 95% CIs of
  ±0.003–0.004 F1 for the four audited models — their own ranking is solid and
  threshold-robust (±100 meV/atom sweep) — but **43 of 59 adjacent pairs on the full
  60-model leaderboard are separated by less than one CI width**, so the published
  ordering at the top is unlikely to be statistically resolvable. The three
  MPtrj-trained models share failure modes (error correlation r ≈ 0.76; joint-miss
  rate 25× the independence prediction), while ORB v2 contributes 1,886 unique
  true-positive discoveries vs ~350–390 each for the rest — a quantified case that
  training-data diversity, not just leaderboard rank, drives discovery value.
- **Ground-truth audit: the benchmark's e_above_hull column is not
  pymatgen-version-independent.** Recomputing it from the published structure
  entries + MP2020 corrections + the published 2023 phase diagram reproduces 497/500
  subset values to ≤0.001 meV/atom under pymatgen 2026, but 3/500 shift by
  119–217 meV/atom and 2 stability labels flip — traced to MP2020 anion-correction
  assignment (oxidation-state guessing heuristics) drifting across pymatgen
  versions. ~0.5% label ambiguity is material when adjacent leaderboard models are
  separated by ΔF1 ≈ 0.001–0.003.

## Reproducibility findings

- Figshare landing URLs are WAF-blocked for naive downloaders (empty body on plain
  GET); the API endpoint works.
- The PyPI wheel and the GitHub repo ship **different module layouts under the same
  version string** (1.3.1), so `pip install` alone does not reproduce the repo's
  evaluation entrypoints.
- The upstream md5 for the WBM initial-structures file is **stale** (it belongs to an
  older artifact in the same Figshare article) — and the upstream download path never
  verifies md5, so only independent auditors hit it.
- Dependency drift (ruamel.yaml 0.19 + current pymatgen/monty) breaks pymatgen config
  loading with an opaque error; pinning `<0.19` fixes it.

## Why it matters

This is a concrete, command-level audit of whether AI-for-science benchmark claims
are reproducible from public code, data, and model execution — not just whether the
numbers are internally consistent. The positive result is itself informative
(Matbench Discovery's pipeline held up well), and the findings show where independent
verification breaks even when the science is sound.

## Limitations

- Layer B covers three models (CHGNet, MACE-MP-0, and ORB v2) on 500 of 257k
  structures — a small deterministic subset; not a full leaderboard regeneration.
- Only the discovery task was audited (not phonons, geometry, or diatomics).
- Single environment: Windows 11, Python 3.11, RTX 4090.
